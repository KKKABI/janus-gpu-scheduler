# 7.16 实验汇报总结

## 一、方向4 多 Graph 并发（收尾）

### 做了哪些

- 补全所有模型的双客户端测试和单客户端 scale 测试
- 修正统计口径：串行总延迟=加和，并发=max
- 测试 sm_fraction 0.5 vs 1.0 对所有模型的影响
- 测试 GoogLeNet、ConvNeXt、NASNet 的 scale 扩展性

### 结论

| 模型类型 | 代表 | 并发表现 |
|---------|------|---------|
| 轻 | MobileNetV2、GoogLeNet | 完美并发，单延迟不变 |
| 中 | ResNet50、ConvNeXt | N=2 开始退化 |
| 重 | NASNet | N=2 即退化 1.84x |

sm_fraction 改 profile 资源不能解决多 Graph 争抢。该方向可收尾。

## 二、方向1 HP-HP Interference（主力）

### 实验一：候选组合统计

在 `schedule()` 每次调用时统计不同 α 阈值下的可行候选组合数。

**结论**：α=0.9 时 62-78% 调度轮次仅 1 个候选，方向 B 的余弦相似度选无可选。

### 实验二：根因分析

通过 ConvNeXt profile 分析发现：elementwise kernel grid=1568 block，但 VirtualSM 要求所有 block 同时驻留 → 64 SM × 12 = 768 上限 → 塞不下 → 判不可行。大量组合被误杀。

### 实验三：时域仿真修复

修改 `can_apply_launch()`，推进时间释放已完成 block 再重试，不再要求同时驻留。

| Model | 修复前 | 修复后 | 改善 |
|-------|-------|-------|------|
| Inception-v3 | 2.237ms | 2.002ms | **-10.5%** |
| NASNet | 8.873ms | 8.078ms | **-9.0%** |
| GoogLeNet | 0.804ms | 0.789ms | -1.9% |
| 其余四模型 | 无变化 | 无变化 | — |

### 实验四：三策略对比

在时域仿真基础上，对比 max_occupancy（纯占用率）、cosine（方向 B）、min_resource（资源加和均衡）三种选择策略。全七模型、α=0.2~0.9 sweep。

**结论**：所有模型所有 α 下，三种策略差异 < 0.07ms（噪声等价）。max_occupancy 最简单且 5/7 模型最优。α=0.9 整体最优。

### 方向1 最终方案

**时域仿真 + α=0.9 + max_occupancy**

- 两个创新点：(1) 发现 VirtualSM 过于保守的根因 (2) 时域仿真修复显著提升
- 一个否定发现：余弦相似度和资源加和对选择无效，纯占用率已是最优
